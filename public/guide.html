Quick Setup

Step 1

Deploy the entire database in one command, or run each file individually for fine-grained control.

1
Connect to MySQL
Open a terminal and log in as root (or any user with CREATE DATABASE rights).
# Terminal
mysql -u root -p
2
Run the master script
This sources all files in the correct dependency order.
-- Inside MySQL shell
SOURCE sql/run_all.sql;
✅
On success you will see: Agricultural Services Database deployed successfully.
The seed data loads 4 farmers, 5 gardens, 3 seasons, 2 export batches, and 1 cooperative.
3
Verify the database
Confirm all tables were created and data was loaded.
USE agri_db;
SHOW TABLES;
SELECT COUNT(*) AS Farmers FROM Farmer;
SELECT COUNT(*) AS Gardens FROM Garden;

Schema Demo

DDL + DML

Real insert sequences that walk through the data model from top to bottom.

1
Insert the location hierarchy
Location data must exist before any Person, Farm, or Garden.
INSERT INTO Region (Name) VALUES ('Eastern');
INSERT INTO District (Name, RegionID) VALUES ('Mbale', 1);
INSERT INTO Subcounty (Name, DistrictID) VALUES ('Bufumbo', 1);
INSERT INTO Village (Name, SubcountyID) VALUES ('Nabyeya', 1);
2
Register a new farmer
Person must be inserted before Farmer (NIN FK constraint).
-- Step A: create the supertype Person record
INSERT INTO Person (NIN, FirstName, LastName, Sex, DOB, Phone1, VillageID)
VALUES ('CM95080010X', 'John', 'Wafula', 'M', '1995-08-12', '0703456789', 1);

-- Step B: create the Farm (Acreage starts at 0, trigger will update it)
INSERT INTO Farm (YearPlanted) VALUES (2021);
SET @new_farm = LAST_INSERT_ID();

-- Step C: create the Farmer subtype
INSERT INTO Farmer (NIN, FarmID)
VALUES ('CM95080010X', @new_farm);
SET @new_farmer = LAST_INSERT_ID();

-- Confirm
SELECT f.FarmerID, p.FirstName, p.LastName, fm.Acreage
FROM Farmer f
JOIN Person p  ON f.NIN    = p.NIN
JOIN Farm   fm ON f.FarmID = fm.FarmID
WHERE f.FarmerID = @new_farmer;
3
Add a Coffee Grade and a Garden
Garden links Farm ↔ CoffeeGrade ↔ Village.
-- Coffee classification
INSERT INTO CoffeeType (TypeName, TreeSpacing, PlantsPerAcre)
VALUES ('Arabica', '8×8 ft', 680);

INSERT INTO CoffeeGrade (TypeID, GradeName, MinPriceUGX, MaxPriceUGX)
VALUES (LAST_INSERT_ID(), 'Bugisu AA', 9500, 13000);
SET @grade = LAST_INSERT_ID();

-- Garden (trigger AfterGardenInsert fires, updates Farm.Acreage)
INSERT INTO Garden
  (FarmID, VillageID, Acreage, GradeID, PlantingDate, NumberOfTrees)
VALUES
  (@new_farm, 1, 3.00, @grade, '2021-06-01', 680);

-- Check Farm.Acreage was auto-updated by trigger
SELECT FarmID, Acreage, CoffeeVariety FROM Farm
WHERE  FarmID = @new_farm;
FarmIDAcreageCoffeeVariety
53.0000Bugisu AA
⚡
Trigger fired: AfterGardenInsert + FarmCoffeeVarietyInsert — both ran automatically. Farm.Acreage and CoffeeVariety were set without any explicit UPDATE.

Triggers Demo

12 Triggers
AfterGardenInsert / Update AFTER INSERT + UPDATE on Garden
Recomputes Farm.Acreage = SUM(Garden.Acreage) automatically. Also rebuilds CoffeeVariety string.
BlockUnverifiedGardenHarvest BEFORE INSERT on ProductionRecord
Raises SQLSTATE 45000 if Garden.IsVerified = FALSE. A harvest cannot be recorded until an extension worker verifies the plot.
BeforeDistributionItemInsert BEFORE INSERT on DistributionItem
Checks Inputs.Qty >= QuantityGiven. Blocks if stock is insufficient.
AfterDistributionItemInsert AFTER INSERT on DistributionItem
Deducts Inputs.Qty -= QuantityGiven. Writes an audit row to DistributionLog.
AfterProcessingRecord AFTER INSERT on ProcessingRecord
Calculates LossKg = RawKgIn − ProcessedKgOut and YieldPct. Writes to ProcessingAuditLog.
AfterExportBatchInsert AFTER INSERT on ExportBatch
Upserts SeasonExportSummary running totals (TotalKg, TotalUSD) for ministry dashboards.
▶
Demo — Block unverified garden harvest
Garden 4 has IsVerified = FALSE in the seed data.
-- This will be BLOCKED by trigger T3
INSERT INTO ProductionRecord
  (GardenID, SeasonID, DateHarvested, HarvestedKg, ProductionCost)
VALUES (4, 1, '2025-01-10', 180.00, 900000);

-- Expected: ERROR 1644 (45000):
-- Garden must be verified before recording a harvest.
-- First verify it, then retry
UPDATE Garden SET IsVerified = TRUE WHERE GardenID = 4;

INSERT INTO ProductionRecord
  (GardenID, SeasonID, DateHarvested, HarvestedKg, ProductionCost)
VALUES (4, 1, '2025-01-10', 180.00, 900000);
-- Query OK, 1 row affected
▶
Demo — Automatic stock deduction
After inserting a DistributionItem, Inputs.Qty is automatically reduced.
-- Before: check NPK fertilizer stock
SELECT InputID, Name, Qty FROM Inputs WHERE InputID = 2;
-- Qty: 800

INSERT INTO Distribution
  (FarmerID, GardenID, WorkerID, DateDistributed)
VALUES (1, 1, 1, CURDATE());
SET @d = LAST_INSERT_ID();

INSERT INTO DistributionItem (DistribID, InputID, QuantityGiven)
VALUES (@d, 2, 100.00);

-- After: trigger deducted 100 automatically
SELECT InputID, Name, Qty FROM Inputs WHERE InputID = 2;
-- Qty: 700

-- Verify the audit log was written
SELECT * FROM DistributionLog ORDER BY LogTime DESC LIMIT 1;

Stored Procedures Demo

12 Procedures
1
RegisterFarmer — atomic onboarding
Inserts Person + Farmer in a single transaction. Rolls back if either fails.
CALL RegisterFarmer(
  'CM99010020Y',  -- NIN
  'David',         -- FirstName
  'Kiiza',          -- LastName
  'M',              -- Sex
  '1999-01-15',    -- DOB
  '0712345678',    -- Phone
  2,               -- VillageID
  1,               -- FarmID (existing farm)
  @new_farmer_id   -- OUT: returned FarmerID
);

SELECT @new_farmer_id AS new_farmer;
2
VerifyGarden + RecordHarvest
Verify the garden (extension worker) then record the harvest for the season.
-- Worker 1 verifies Garden 4 (was unverified in seed data)
CALL VerifyGarden(4, 1);

-- Now record the harvest (trigger will no longer block it)
CALL RecordHarvest(
  4,            -- GardenID
  1,            -- SeasonID
  '2025-01-10',-- DateHarvested
  180.00,       -- HarvestedKg
  900000,       -- ProductionCost (UGX)
  @prod_id      -- OUT: ProdID
);

SELECT @prod_id AS new_prod_id;
3
DistributeToGarden — JSON-driven bulk items
Passes multiple input items as a JSON array. Pre-validates stock for all before touching anything.
CALL DistributeToGarden(
  1,    -- FarmerID
  1,    -- GardenID
  1,    -- WorkerID
  '[{"InputID":2,"Qty":50},{"InputID":3,"Qty":30}]',
  @distrib_id
);

-- Verify stock was deducted for BOTH items
SELECT InputID, Name, Qty FROM Inputs
WHERE  InputID IN (2, 3);
4
CreateExportBatch — license + stock validation
Validates exporter license and won't allow exporting more than was processed.
CALL CreateExportBatch(
  1,           -- ExporterID
  1,           -- ProcessID
  1,           -- SeasonID
  4,           -- GradeID (Bugisu AA)
  40.00,       -- QuantityKg
  'Netherlands',-- DestCountry
  3.72,        -- USDPricePerKg
  @batch_id    -- OUT: BatchID
);

SELECT BatchID, QuantityKg, TotalUSD, Status
FROM ExportBatch WHERE BatchID = @batch_id;
⚡
AfterExportBatchInsert trigger fired: SeasonExportSummary totals were updated automatically. TotalUSD is a MySQL generated column (no trigger needed).
5
GetFarmerProductionReport — full history
Returns harvests, processing, and exports for a farmer in one call.
CALL GetFarmerProductionReport(1);
-- Returns: GardenID, SeasonName, DateHarvested, HarvestedKg,
--          ProcessType, RawKgIn, ProcessedKgOut, LossKg,
--          YieldPct, DestCountry, ExportedKg, TotalUSD

Transactions Demo

ACID
ℹ️
All transactions use START TRANSACTION … COMMIT with DECLARE EXIT HANDLER FOR SQLEXCEPTION BEGIN ROLLBACK; RESIGNAL; END; so that any error in any step rolls back the entire operation.
1
TXN-1 — Full Farmer Registration (5 tables)
START TRANSACTION;

INSERT INTO Person (NIN, FirstName, LastName, Sex, DOB, Phone1, VillageID)
VALUES ('CM98051122Z', 'Mary', 'Nabwire', 'F', '1998-05-11', '0756789012', 2);

INSERT INTO Farm (YearPlanted) VALUES (2022);
SET @f = LAST_INSERT_ID();

INSERT INTO Farmer (NIN, FarmID) VALUES ('CM98051122Z', @f);
SET @fa = LAST_INSERT_ID();

INSERT INTO Garden
  (FarmID, VillageID, Acreage, GradeID, PlantingDate, NumberOfTrees)
VALUES (@f, 2, 2.00, 4, '2022-03-01', 545);
SET @g = LAST_INSERT_ID();

INSERT INTO GardenSeason (GardenID, SeasonID) VALUES (@g, 2);

COMMIT;
-- All 5 inserts succeed together, or none do.
2
TXN-2 — Harvest → Process → Sell pipeline
START TRANSACTION;

INSERT INTO ProductionRecord
  (GardenID, SeasonID, DateHarvested, HarvestedKg, ProductionCost)
VALUES (1, 2, '2025-07-15', 200.00, 1000000);
SET @pr = LAST_INSERT_ID();

-- Trigger computes LossKg & YieldPct automatically
INSERT INTO ProcessingRecord
  (GardenID, ProcessorID, SeasonID, RawKgIn, ProcessedKgOut, ProcessType, ProcessedDate)
VALUES (1, 1, 2, 200.00, 140.00, 'Wet', '2025-07-20');

INSERT INTO SaleRecord
  (ProdID, DateSold, QuantityKg, UnitPrice, PaymentStatus, BuyerType)
VALUES (@pr, '2025-07-25', 80.00, 11000.00, 'Paid', 'Exporter');

COMMIT;
3
TXN-6 — Season Closure & Rollover
START TRANSACTION;

-- Close current season
UPDATE Season SET EndDate = CURDATE()
WHERE  SeasonID = 2 AND EndDate IS NULL;

-- Open new season
INSERT INTO Season (SeasonName, StartDate, SeasonOrder)
VALUES ('2026 First Season',
        DATE_ADD(CURDATE(), INTERVAL 30 DAY), 'First');

COMMIT;

User Privileges Demo

RBAC
User Account Password Permissions

1
Deploy user accounts
-- (Already included in run_all.sql)
SOURCE sql/05_users.sql;
2
Test the extension worker role
Connect as agri_extworker and prove the permission boundaries.
# Connect as extension worker
mysql -u agri_extworker -p'ExtWork@2026!' agri_db

-- Allowed: view assigned gardens
SELECT * FROM Farmer_Garden_Overview LIMIT 5;

-- Allowed: record a farm visit
INSERT INTO FarmVisit
  (WorkerID, FarmerID, GardenID, VisitDate, Purpose)
VALUES (1, 1, 1, CURDATE(), 'Monitoring');

-- BLOCKED: cannot view SaleRecord (financial data)
SELECT * FROM SaleRecord;
-- ERROR 1142: SELECT command denied

-- BLOCKED: cannot run ministry reporting
CALL GetSeasonExportSummary(1);
-- ERROR 1370: execute command denied
3
Test the read-only auditor role
# Connect as auditor
mysql -u agri_readonly -p'ReadOnly@2026!' agri_db

-- Allowed: read any view
SELECT * FROM Ministry_Export_Summary;
SELECT * FROM Processing_Yield_Report;

-- BLOCKED: no base table access at all
SELECT * FROM Person;
-- ERROR 1142: SELECT command denied
4
Verify all grants from admin
